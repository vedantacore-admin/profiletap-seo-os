## Mandatory Page Elements By Type

### Homepage

- clear smart identity platform positioning
- full feature inventory listed once
- use-case overview
- product capability blocks
- primary conversion CTA

### Solution hub page

- hub-specific positioning
- relevant embedded feature subset only
- category or audience-specific workflows
- CTA into the hub's main conversion path

### Category page

- definition of the category
- benefits and use cases
- product walkthrough
- CTA to create profile or request demo

### Use-case page

- audience-specific pain points
- segment-specific use cases
- tailored feature framing inherited from the relevant hub
- segment-relevant CTA

### Comparison page

- competitor alternative framing
- comparison table
- reasons to switch
- FAQ covering migration or switching concerns

### Blog page

- direct answer to the query
- educational depth
- internal links to commercial pages
- CTA block that routes readers to the correct product page

## Notes For Writer

- avoid generic NFC-only positioning unless the keyword is specifically NFC-led
- keep ProfileTap framed as a broader smart identity platform
- do not turn embedded features into standalone page intent unless the SEO system explicitly maps that feature to its own page later
- convert internal feature tokens like `qr_sharing` into natural labels like `QR sharing` in final copy
- if the page is India-first, use Indian business language where natural
- do not target a second primary intent on the same page
