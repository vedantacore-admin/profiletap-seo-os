# Content Optimization & On-Page SEO

> Claude Code Skill: Complete on-page SEO and content optimization framework for any website or page type.
> Use this as a reference when creating new content, auditing existing pages, or building content briefs.

---

## 1. Title Tag Formulas by Page Type

Title tags are the single highest-impact on-page element. Maximum 60 characters displayed; Google may rewrite if poorly constructed.

### 1.1 Homepage
```
[Brand Name] — [Primary Value Proposition or Tagline]
[Brand Name]: [Category/What You Do]
[Primary Keyword] | [Brand Name]
```
Examples:
- "Acme CRM — Sales Pipeline Software for Growing Teams"
- "FreshBooks: Online Accounting Software for Small Business"

### 1.2 Category / Service Page
```
[Primary Keyword] — [Differentiator] | [Brand]
[Modifier] [Primary Keyword] for [Audience] | [Brand]
Best [Primary Keyword] — [Benefit] | [Brand]
```
Examples:
- "Project Management Software — Built for Remote Teams | Acme"
- "Email Marketing Tools for E-Commerce | MailPro"

### 1.3 Product / Feature Page
```
[Product Name] — [Key Benefit/Feature] | [Brand]
[Primary Keyword]: [Product Name] by [Brand]
```
Examples:
- "Kanban Boards — Visual Project Tracking | Acme PM"
- "Invoice Generator: Create Professional Invoices Free | FreshBooks"

### 1.4 Comparison Page
```
[Product A] vs [Product B]: [Year] Comparison | [Brand]
[Product A] vs [Product B] — Which Is Better for [Use Case]?
Compare [Category]: [Product A] vs [Product B] vs [Product C]
```
Examples:
- "Asana vs Monday.com: 2026 Feature Comparison | PMReview"
- "Mailchimp vs ConvertKit — Which Is Better for Creators?"

### 1.5 Blog / Informational Page
```
[How to / What Is / Guide to] [Primary Keyword] ([Year])
[Number] [Primary Keyword] [Tips/Strategies/Examples] for [Year]
[Primary Keyword]: [Subtitle That Adds Context]
```
Examples:
- "How to Build a Content Calendar (2026 Guide)"
- "15 Email Subject Line Formulas That Increase Open Rates"

### 1.6 Title Tag Rules
- Place the primary keyword as close to the beginning as possible.
- Include the brand name (usually at the end, separated by | or —).
- Keep under 60 characters to avoid truncation.
- Every page on the site must have a unique title tag.
- Do not keyword-stuff; one primary keyword plus one secondary is the max.
- Include the year only on content that is genuinely updated annually.

## 2. Meta Description Templates

Meta descriptions do not directly affect rankings but heavily influence click-through rate. Maximum 155 characters displayed.

### 2.1 Templates by Intent

**Informational:**
```
Learn [what the content covers]. This guide covers [key topics]. [CTA: Read now / Get started / See examples].
```

**Commercial:**
```
Compare the best [category]. We reviewed [number] options to help you find the right [product type] for [audience/need]. See our top picks.
```

**Transactional:**
```
[Action verb] [product/service] starting at [price/free]. [Key benefit]. [Social proof if space allows]. [CTA: Start free trial / Sign up today].
```

**Local:**
```
[Service] in [City]. [Key differentiator]. [Social proof: X years, Y clients]. Call [phone] or book online today.
```

### 2.2 Meta Description Rules
- Include the primary keyword naturally (Google bolds matching terms).
- Include a clear call-to-action.
- Convey a unique value proposition — why click this result over others?
- Never duplicate meta descriptions across pages.
- If you cannot write a compelling description, leave it blank and let Google auto-generate (better than a bad meta description).
- Use active voice and direct language.

## 3. H1 and Header Hierarchy

### 3.1 H1 Rules
- Every page must have exactly one H1 tag.
- The H1 should contain the primary keyword (or a close variant).
- The H1 can differ from the title tag — use it to be more descriptive or conversational.
- The H1 should be the first heading visible on the page.

### 3.2 Header Hierarchy Best Practices
```
H1: Primary topic / page title (one per page)
  H2: Major section headings (primary subtopics)
    H3: Subsections within an H2
      H4: Sub-subsections (use sparingly)
```

**Rules:**
- Never skip heading levels (do not go from H2 to H4).
- Use H2s for scannable section breaks — aim for one H2 every 200-300 words.
- Include secondary keywords in H2s naturally.
- Use question-format H2s for FAQ/PAA targeting (e.g., "How much does X cost?").
- Do not use headings for styling purposes — use CSS instead.
- Sidebar headings, footer headings, and widget headings should not use H1-H3 tags.

### 3.3 Header Keyword Mapping
| Heading Level | Keyword Strategy |
|--------------|-----------------|
| H1 | Primary keyword or close variant |
| H2 | Secondary keywords, one per section |
| H3 | Long-tail variants, question keywords, supporting terms |
| H4+ | Organizational only — no keyword targeting needed |

## 4. URL Structure Rules

### 4.1 URL Best Practices
- Use lowercase letters only.
- Use hyphens (-) to separate words. Never underscores, spaces, or camelCase.
- Keep URLs as short as possible while remaining descriptive (3-5 words ideal).
- Include the primary keyword in the URL slug.
- Remove stop words (a, an, the, of, in, for) unless they are essential for clarity.
- Use a logical folder hierarchy that mirrors site architecture.
- Never include dates in evergreen content URLs (avoid /2026/03/keyword/).
- Never include URL parameters for content pages (parameters are for filtering/tracking).

### 4.2 URL Hierarchy Examples
```
Homepage:         /
Category:         /email-marketing-software/
Subcategory:      /email-marketing-software/automation/
Product/Feature:  /email-marketing-software/automation/drip-campaigns/
Comparison:       /compare/mailchimp-vs-convertkit/
Blog post:        /blog/email-marketing-tips/
Resource:         /resources/email-templates/
```

### 4.3 URL Changes
- Never change a URL without implementing a 301 redirect from the old URL.
- Audit internal links pointing to the old URL and update them.
- Update the XML sitemap.
- Monitor Google Search Console for crawl errors after the change.

## 5. Content Structure by Page Type

### 5.1 Homepage
- Hero section: Clear value proposition, primary CTA, trust signals.
- Problem/solution section: What pain point do you solve?
- Features/benefits overview (link to detailed pages).
- Social proof: testimonials, logos, case study highlights.
- Secondary CTA or lead magnet.
- Brief "How it works" section if the product needs explanation.
- Target word count: 800-1,500 words of indexable content.

### 5.2 Category / Service Page
- H1 with primary keyword.
- Introductory paragraph (100-150 words) explaining the category.
- Feature/benefit grid or list.
- Comparison table if multiple products/plans exist.
- FAQ section (3-5 questions).
- Internal links to subcategory or individual product pages.
- CTA blocks interspersed throughout.
- Target word count: 1,000-2,500 words.

### 5.3 Product / Feature Page
- H1: Product/feature name with benefit.
- Hero: Screenshot or demo, primary CTA.
- Key benefits (3-5) with supporting details.
- How it works walkthrough.
- Use cases or audience segments.
- Integration or compatibility info.
- Pricing summary or link to pricing.
- Testimonials specific to this product.
- Target word count: 800-2,000 words.

### 5.4 Comparison Page
- H1: "[A] vs [B]" or "Best [Category]: [A] vs [B] vs [C]."
- TL;DR summary at the top (who should pick what).
- Side-by-side feature comparison table.
- Detailed analysis of each option (200-400 words per option).
- Pricing comparison.
- Pros/cons for each.
- Final verdict with recommendation.
- Target word count: 2,000-4,000 words.

### 5.5 Blog / Informational Page
- H1: Descriptive, keyword-rich, benefit-oriented.
- Introduction: Hook, context, what the reader will learn (100-150 words).
- Table of contents for posts over 1,500 words.
- Logical H2/H3 structure covering all subtopics.
- Actionable advice, not just definitions.
- Examples, screenshots, data, or case studies.
- Internal links to related content and relevant product pages.
- Conclusion with summary and next step/CTA.
- Target word count by intent:
  - Definitional ("what is X"): 800-1,200 words.
  - How-to guide: 1,500-3,000 words.
  - Ultimate guide: 3,000-5,000 words.
  - Listicle: 1,500-3,000 words.

## 6. E-E-A-T Signals Implementation

E-E-A-T = Experience, Expertise, Authoritativeness, Trustworthiness. Critical for YMYL (Your Money or Your Life) topics, important for all content.

### 6.1 Experience Signals
- Include first-hand observations, original screenshots, personal results.
- Use phrases like "In our experience," "When we tested this," "After implementing X, we found."
- Show original data, case studies, or proprietary research.

### 6.2 Expertise Signals
- Display author name, bio, and credentials on every article.
- Link to the author's LinkedIn, portfolio, or relevant profiles.
- Create dedicated author pages with full bios and article archives.
- Use expert quotes or cite primary sources.
- Demonstrate depth (not just surface-level coverage).

### 6.3 Authoritativeness Signals
- Earn backlinks from authoritative sites in the niche.
- Get mentioned or cited by other experts.
- Publish on the topic consistently over time (topical authority).
- Display relevant awards, certifications, or partnerships.

### 6.4 Trustworthiness Signals
- HTTPS (non-negotiable).
- Clear contact information (physical address, phone number, email).
- Privacy policy, terms of service.
- Accurate, up-to-date information.
- Cite sources with outbound links to authoritative references.
- Display reviews, testimonials, and trust badges.
- No deceptive practices (hidden text, cloaking, misleading claims).

## 7. Keyword Density and Semantic Coverage

### 7.1 Keyword Density Guidelines
- There is no magic keyword density number. Over-optimization is penalized.
- Rule of thumb: Primary keyword should appear naturally 3-8 times in a 1,500-word article.
- Include the primary keyword in: H1, first 100 words, at least one H2, last 100 words, and alt text of the primary image.
- Use variations, synonyms, and related terms throughout (semantic SEO).

### 7.2 Semantic Coverage (NLP/LSI)
- Analyze the top 5 ranking pages for your target keyword.
- Identify commonly used terms, phrases, and topics that appear across all top results.
- Your content should cover the same semantic territory.
- Tools: Surfer SEO, Clearscope, MarketMuse, Frase — these provide NLP-based content scores.
- Manual method: Read top 5 results, extract all subtopics covered, ensure your content addresses each one.

### 7.3 Semantic Coverage Checklist
- [ ] Primary keyword appears in H1, intro, and conclusion.
- [ ] 3-5 secondary keywords appear in H2s.
- [ ] Related entities and concepts are mentioned naturally.
- [ ] No important subtopic from the top-ranking pages is missing from your content.
- [ ] Synonyms and variants are used instead of repeating the exact same phrase.

## 8. Internal Linking Placement Rules

### 8.1 Link Placement Strategy
- Include 3-5 internal links per 1,000 words of content.
- Place the most important internal link in the first 200 words.
- Use descriptive anchor text that includes the target page's primary keyword.
- Never use "click here" or "read more" as anchor text.
- Link to relevant product/service pages from informational content (bridge informational and transactional intent).
- Link to related blog posts to keep users on the site.
- Link to parent category pages from child pages (reinforce hierarchy).

### 8.2 Internal Linking Architecture
| From Page Type | Link To | Purpose |
|---------------|---------|---------|
| Blog post | Related blog posts | Topical clustering |
| Blog post | Product/service pages | Drive commercial traffic |
| Category page | Subcategory / product pages | Navigation and authority flow |
| Product page | Comparison pages | Help decision-making |
| Product page | Related products | Cross-sell |
| Any page | Pillar/hub content | Reinforce topical authority |

### 8.3 Anchor Text Rules
- Use the target page's primary keyword as anchor text (vary slightly to avoid over-optimization).
- Mix exact match (30%), partial match (40%), and branded/natural (30%) anchors.
- Never link the same anchor text to two different pages.

## 9. FAQ Schema Opportunities

### 9.1 When to Add FAQ Schema
- The page naturally addresses common questions about the topic.
- The keyword triggers People Also Ask boxes in the SERP.
- The page is a category, product, or service page (not a pure FAQ page — those use QAPage schema).
- You have 3-8 genuine questions with concise, helpful answers.

### 9.2 FAQ Content Rules
- Questions must be real questions users ask (validate with PAA, forums, support tickets).
- Answers should be 40-80 words each — concise but complete.
- Do not use FAQ schema on every page — Google may ignore excessive use.
- FAQ answers can contain links (including internal links) — use this strategically.
- Markup must match the visible content on the page exactly.

### 9.3 FAQ Schema JSON-LD Template
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is [topic]?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Concise answer to the question, 40-80 words."
      }
    },
    {
      "@type": "Question",
      "name": "How much does [topic] cost?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Answer with pricing details or ranges."
      }
    }
  ]
}
```

## 10. Content Length Guidelines by Intent

| Intent | Recommended Length | Rationale |
|--------|-------------------|-----------|
| Transactional (product page) | 800-2,000 words | Users want to buy, not read essays. Enough info to convert. |
| Commercial (comparison/review) | 2,000-4,000 words | Users need comprehensive evaluation to make a decision. |
| Informational (how-to) | 1,500-3,000 words | Enough depth to fully answer the question. |
| Informational (ultimate guide) | 3,000-5,000 words | Comprehensive coverage builds authority and earns links. |
| Informational (definition) | 600-1,200 words | Quick, clear answer. Over-padding hurts user experience. |
| Navigational | 500-1,000 words | Users know where they want to go. Minimal content needed. |
| Local/geo landing page | 800-1,500 words | Enough for unique, localized content without padding. |

**Important:** Always check what the top-ranking pages are doing. If the top 5 results average 3,000 words, aim for 3,000+ words with better quality. If they average 800 words, do not write 3,000 words of filler.

## 11. Readability and UX Writing Standards

### 11.1 Readability Targets
- Aim for a Flesch-Kincaid grade level of 6-8 for general audiences.
- Use short sentences (15-20 words average).
- Use short paragraphs (2-4 sentences max).
- Use bullet points and numbered lists for scannable content.
- Use bold text to highlight key phrases and takeaways.
- Use subheadings (H2/H3) every 200-300 words as visual breaks.

### 11.2 UX Writing Rules
- Front-load important information (inverted pyramid style).
- Use active voice over passive voice.
- Address the reader directly ("you" not "one" or "users").
- Define jargon on first use.
- Use transition words to connect sections.
- Include a TL;DR or key takeaway box for long content.

## 12. Image Optimization

### 12.1 Alt Text Formulas
- **Product image:** "[Product Name] — [Key Feature or Use Case]"
- **Screenshot:** "Screenshot of [what the screenshot shows]"
- **Infographic:** "[Topic] infographic showing [what it illustrates]"
- **Stock/decorative:** Leave alt text empty (alt="") for purely decorative images.
- **Chart/graph:** "[Metric] by [dimension] — [key takeaway]"

### 12.2 File Naming
- Use descriptive, keyword-rich file names: `project-management-dashboard.webp` not `IMG_3847.jpg`.
- Use hyphens between words.
- Keep file names under 50 characters.

### 12.3 Image Compression and Format
| Format | Best For | Target |
|--------|----------|--------|
| WebP | All web images (preferred) | 80% quality, lossy |
| AVIF | Modern browsers, best compression | 70% quality |
| PNG | Screenshots, images requiring transparency | Compress with TinyPNG |
| SVG | Icons, logos, simple graphics | Minify with SVGO |
| JPEG | Fallback for older browsers | 80% quality |

- Serve images in next-gen formats (WebP/AVIF) with JPEG fallback.
- Use responsive images (srcset) to serve appropriate sizes per viewport.
- Lazy load images below the fold.
- Set explicit width and height attributes to prevent CLS.

## 13. Featured Snippet Optimization

### 13.1 Snippet Types and How to Win Them
| Snippet Type | Trigger | How to Optimize |
|-------------|---------|-----------------|
| **Paragraph** | "What is," "Why," definitional queries | Provide a 40-60 word answer directly below the H2 that poses the question. |
| **List (ordered)** | "How to," "Steps to," process queries | Use an ordered list (H2 for the question, H3s or ol/li for each step). |
| **List (unordered)** | "Best," "Top," "Types of" | Use bullet points under the H2 question. Keep items concise. |
| **Table** | Comparison queries, data queries | Use an HTML table with clear headers and concise cell content. |

### 13.2 Snippet Optimization Rules
- You must already rank on page 1 (positions 1-10) to be eligible for featured snippets.
- Structure the answer immediately after the triggering H2 question.
- Keep paragraph snippets to 40-60 words.
- Keep list snippets to 5-8 items.
- Use the exact question phrasing in your H2 when targeting a PAA or snippet.
- Include a brief answer AND a more detailed explanation below it.

## 14. Output Format: On-Page SEO Audit Checklist

Use this checklist when auditing any page on any website.

### Title Tag
- [ ] Contains primary keyword near the beginning.
- [ ] Under 60 characters.
- [ ] Unique (not duplicated on any other page).
- [ ] Includes brand name.
- [ ] Compelling enough to earn clicks.

### Meta Description
- [ ] Contains primary keyword.
- [ ] Under 155 characters.
- [ ] Includes a call-to-action.
- [ ] Unique (not duplicated on any other page).
- [ ] Communicates clear value proposition.

### URL
- [ ] Contains primary keyword.
- [ ] Uses hyphens, lowercase only.
- [ ] Short and descriptive (3-5 words in slug).
- [ ] No parameters, dates, or unnecessary folders.

### Headings
- [ ] Exactly one H1 containing primary keyword.
- [ ] H2s used for major sections (contain secondary keywords).
- [ ] No heading levels skipped.
- [ ] Headers used for structure, not styling.

### Content
- [ ] Primary keyword in first 100 words.
- [ ] Primary keyword in conclusion.
- [ ] Secondary keywords used naturally in body.
- [ ] Semantic coverage matches top-ranking competitors.
- [ ] Content length appropriate for intent and competitive landscape.
- [ ] Content is original (not duplicated from other pages).
- [ ] Readability appropriate for target audience.
- [ ] Content is up-to-date and accurate.

### Internal Links
- [ ] 3-5 internal links per 1,000 words.
- [ ] Descriptive anchor text used.
- [ ] Links to relevant product/service pages from informational content.
- [ ] Links to related content for topical clustering.
- [ ] No broken internal links.

### Images
- [ ] All images have descriptive alt text (except decorative).
- [ ] Images use next-gen formats (WebP/AVIF).
- [ ] Images are compressed.
- [ ] Images have explicit width/height.
- [ ] Images below the fold are lazy loaded.

### Schema Markup
- [ ] Appropriate schema type implemented (see Technical SEO skill).
- [ ] FAQ schema added if the page answers common questions.
- [ ] Schema validates in Google Rich Results Test.

### E-E-A-T
- [ ] Author name and bio visible.
- [ ] Sources cited with outbound links.
- [ ] Original insights, data, or experience demonstrated.
- [ ] Content reviewed and updated date displayed.

### User Experience
- [ ] Content is scannable (short paragraphs, lists, bold text).
- [ ] CTA is clear and visible.
- [ ] No intrusive interstitials or popups.
- [ ] Mobile-friendly layout.
